Интерактивный `PowerShell`-мастер для `Windows`, который экспортирует и импортирует пользовательские настройки `Adobe` между профилями системы.

Скрипт рассчитан на ручной перенос настроек, а не на полный побайтный клон рабочего окружения.

## Возможности

- находит локальные профили `Windows`
- показывает поддерживаемые приложения и найденные версии
- управляется стрелками, `Enter`, `Space` и `Esc`
- экспортирует настройки в папку и `zip`
- импортирует настройки в выбранный профиль
- делает резервную копию перед заменой файлов
- умеет перезапускаться с правами администратора, если выбран чужой профиль

## Поддерживаемые приложения

- `Premiere Pro`
- `After Effects`
- `Photoshop`
- `Adobe Media Encoder`

## Требования

- `Windows 10` или `Windows 11`
- `Windows PowerShell 5.1` или `pwsh 7+`
- запуск в обычной консоли `PowerShell`, а не в `ISE`
- закрытые приложения `Adobe` перед импортом

## Быстрый старт

```powershell
powershell -ExecutionPolicy Bypass -File .\adobe-settings-manager.ps1
```

После запуска мастер:

1. предлагает выбрать `Экспорт` или `Импорт`
2. показывает найденные профили `Windows`
3. предлагает выбрать приложения
4. показывает подтверждение перед выполнением
5. выполняет операцию и пишет журнал

## Куда сохраняются данные

Экспорт по умолчанию:

```text
C:\Users\<пользователь_запуска>\Desktop\AdobeSettingsExports\
```

Резервные копии перед импортом:

```text
C:\Users\<пользователь_запуска>\Desktop\AdobeSettingsBackups\
```

Если `Desktop` недоступен, скрипт использует запасной путь через профиль пользователя.

## Содержимое экспортного пакета

Экспорт создаёт папку и `zip` со структурой такого вида:

```text
AdobeSettings-<user>-<timestamp>\
  manifest.json
  operation.log
  payload\
```

Где:

- `manifest.json` содержит пользователя, приложения, пути, список скопированных и пропущенных элементов
- `operation.log` содержит журнал выполнения
- `payload\` хранит экспортированные каталоги настроек с относительными путями внутри профиля

## Что переносится и что не переносится

Инструмент ориентирован именно на настройки пользователя. Поэтому часть элементов специально исключается:

- кэши
- журналы
- временные файлы
- часть машинозависимых файлов

Это нормально для сценария переноса настроек, но не даёт полной гарантии идентичного окружения на новом ПК.

Обычно отдельно могут потребоваться:

- плагины
- шрифты
- внешние `LUT`
- кодеки
- сторонние пресеты вне профиля пользователя

## Структура репозитория

```text
README.md
.gitignore
```

## Ограничения

- это не полный снимок системы
- результат зависит от версий приложений `Adobe` на обеих машинах
- для чужого профиля может понадобиться запуск с правами администратора
- итоговый перенос желательно проверять на целевой машине запуском каждого приложения
